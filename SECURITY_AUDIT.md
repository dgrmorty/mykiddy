## ⚠️ Проблемы безопасности:

### 1. **CORS открыт для всех** 🔴 КРИТИЧНО
**Проблема:** `server.js` использует `app.use(cors())` без ограничений
**Риск:** Любой сайт может делать запросы к вашему API

**Решение:**
```javascript
// В server.js замените:
app.use(cors());

// На:
app.use(cors({
  origin: [
    'https://mykiddy-production.up.railway.app',
    'http://localhost:5174',
    'http://localhost:5173',
    'capacitor://localhost', // Для Android
    'ionic://localhost'      // Для Capacitor
  ],
  credentials: true
}));
```

### 2. **Нет rate limiting** 🟡 СРЕДНЕ
**Проблема:** Нет ограничения на количество запросов к AI API
**Риск:** Злоумышленник может спамить запросами и сжечь ваш API лимит

**Решение:** Добавить rate limiting (например, `express-rate-limit`)

### 3. **Нет валидации на сервере** 🟡 СРЕДНЕ
**Проблема:** Сервер принимает любые данные без проверки
**Риск:** Можно отправить огромные строки, что может вызвать проблемы

**Решение:** Добавить валидацию длины и формата на сервере

### 4. **Admin email хардкоден** 🟢 НИЗКО
**Проблема:** `ADMIN_EMAILS` в `AuthContext.tsx` захардкожен
**Риск:** Низкий, но лучше вынести в конфиг

### 5. **Нет защиты от перебора паролей** 🟡 СРЕДНЕ
**Проблема:** Supabase Auth имеет базовую защиту, но можно добавить дополнительную
**Риск:** Брутфорс атаки на аккаунты

# Деплой и переменные среды

#deploy #env #vercel #railway

---

## Фронтенд (Vite)

| Переменная | Назначение |
|------------|------------|
| `VITE_SUPABASE_URL` | URL проекта Supabase |
| `VITE_SUPABASE_ANON_KEY` | Anon (публичный) ключ |
| `VITE_API_URL` | База для API (Railway), без завершающего `/`; если пусто — тот же origin |
| `VITE_ADMIN_EMAILS` | Список email админов через запятую |

Сборка: `npm run build` → каталог **`dist/`**.

---

## Сервер (Railway / Node)

| Переменная | Назначение |
|------------|------------|
| `PORT` | Порт Express (по умолчанию 3000 локально) |
| `GEMINI_API_KEY` / `API_KEY` / `GOOGLE_*` | Ключ Gemini (см. нормализацию в `server.js`) |
| `SUPABASE_URL`, `SUPABASE_ANON_KEY` | Для учёта `ai_usage` с JWT пользователя |
| `AI_DAILY_TUTOR_LIMIT`, `AI_DAILY_HOMEWORK_LIMIT` | Дневные лимиты |

---

## Supabase

- Миграции: **`supabase/migrations/`** — источник истины.
- Применение к облаку: **`supabase db push --linked`** (нужен CLI и линк проекта).
- Полный **`supabase db dump`** локально может требовать **Docker**.
- Для Google OAuth см. отдельную заметку: [[Operations/Google-OAuth-и-auth-диагностика]].

---

## Типичный прод

- **Vercel** (или аналог): статика SPA + env `VITE_*`.
- **Railway**: `server.js` + env сервера; CORS настроен под фронт.

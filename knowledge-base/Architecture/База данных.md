# База данных

**Провайдер:** Supabase (PostgreSQL + Auth + RLS)

## Схема таблиц

### `profiles` — профили пользователей
Расширяет стандартную `auth.users` Supabase.

| Колонка | Тип | Описание |
|---------|-----|---------|
| `id` | uuid (PK, FK auth.users) | ID пользователя |
| `name` | text | Отображаемое имя |
| `role` | text | 'Student' / 'Parent' / 'Teacher' / 'Admin' / 'Guest' |
| `avatar` | text | URL аватара |
| `xp` | integer | Очки опыта |
| `level` | integer | Текущий уровень |
| `is_approved` | boolean | Одобрён ли аккаунт |
| `streak_current` | integer | Текущий стрик (дней) |
| `streak_longest` | integer | Лучший стрик |

---

## RLS (Row Level Security)

- **Включён** на всех таблицах
- Гость → только чтение публичных курсов
- Ученик → чтение курсов, запись в user_progress и community_posts
- Admin → полный доступ к courses, modules, lessons, schedule_events, settings
- RPC-функции проверяют роль через JWT

⚠️ Критично: admin-операции закрыты политиками Supabase, не только роутингом фронта.

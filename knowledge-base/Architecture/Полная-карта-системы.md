## 1. Слои (сверху вниз)

```
┌─────────────────────────────────────────────────────────────────┐
│ Клиент: браузер (Vite SPA) или Android WebView (Capacitor)       │
│  React 18 · TS · Tailwind · react-router-dom v6                  │
│  Единый Layout → Outlet (страницы в views/)                      │
└───────────────┬───────────────────────────────┬──────────────────┘
                │ HTTPS                         │ HTTPS + JWT
                ▼                               ▼
┌───────────────────────────┐     ┌────────────────────────────────┐
│ Supabase (публичный URL)   │     │ Backend: Express (server.js)   │
│  PostgREST + Auth + RLS    │     │  Railway / локально :3000      │
│  Анон ключ в VITE_*        │     │  Gemini API key только здесь   │
│  Таблицы, RPC, Storage     │     │  /api/ai-tutor, check-homework │
└───────────────────────────┘     │  ai_usage с JWT пользователя   │
                │                 └────────────────────────────────┘
                │
                ▼
         PostgreSQL + политики RLS
         Функции SECURITY DEFINER где нужен обход узких политик
```

## 2. Два «источника правды» для данных

| Источник | Что даёт | Когда |
|----------|----------|--------|
| **Supabase напрямую** | Курсы, прогресс, профили, друзья, витрина (частично) | SPA с `supabase-js`, сессия в браузере |
| **Express** | Вызовы **Gemini**, учёт **ai_usage**, лимиты по дню | Только через `getApiUrl('api/...')` + Bearer JWT |

**Запрещено:** ключ Gemini на клиенте или прямой вызов Gemini из браузера.

---

## 3. Аутентификация и роли

- Сессия: **Supabase Auth** (JWT в клиенте).
- **Guest** в приложении — локальное состояние `Role.GUEST`, не полноценный anon JWT для всех экранов; защищённые роуты → редирект на `/`.
- Роль **Admin** на фронте: `VITE_ADMIN_EMAILS` + нормализация в `AuthContext` (см. [[Roles/Роли пользователей]]).
- **StaffRoute** (`/admin`): только `user.role === Admin`.

---

## 4. Контент обучения

- Иерархия: **Course → Module → Lesson** (таблицы `courses`, `modules`, `lessons`).
- Загрузка списка курсов: **`contentService.getCourses`** → кэш TTL ~90 с → мерж с **`user_progress`** для `isCompleted`.
- Прогресс пишется в **`user_progress`**; **XP** — поля в **`profiles`** + RPC **`increment_xp`** / логика на клиенте (см. миграции).

---

## 5. AI (кросс-срез)

- **Тьютор:** `geminiService.askAiTutor` → `POST /api/ai-tutor` (контекст урока + вопрос).
- **Проверка ДЗ:** `checkHomework` → `POST /api/check-homework` (текст/медиа).
- Лимиты: таблица **`ai_usage`** (по JWT на сервере), env `AI_DAILY_TUTOR_LIMIT`, `AI_DAILY_HOMEWORK_LIMIT`.
- Квота в UI: `GET /api/ai-usage`.

---

## 6. Сообщество и витрина

- **Друзья:** `friendships`.
- **Лента проектов:** `project_posts` (модерация pending/approved/rejected), **`project_post_likes`**, RPC **`list_showcase_authors`**, **`list_approved_showcase_posts`**, **`showcase_like_counts`** — см. [[Architecture/RLS-RPC-и-безопасность]], [[Features/Витрина проектов]].

---

## 7. Мобильное приложение

Отдельный репозиторий/папка **`mykiddymobile`** (Expo): те же Supabase + те же API путей к Railway. Не дублировать бизнес-логику расхоже с вебом без причины.

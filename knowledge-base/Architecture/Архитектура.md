## Общая схема

```
┌─────────────────────────────────────────────┐
│              БРАУЗЕР / ANDROID               │
│                                             │
│  React SPA (Vite)                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ contexts │  │  views   │  │components│  │
│  │ Auth     │  │Dashboard │  │ Layout   │  │
│  │ Content  │  │CourseDetail│ │ ...      │  │
│  │ Branding │  │Profile   │  └──────────┘  │
│  │ Toast    │  │Admin...  │               │
│  └────┬─────┘  └──────────┘               │
│       │                                     │
└───────┼─────────────────────────────────────┘
        │
        ├──────────────────────────────────────▶ Supabase
        │   (auth, courses, progress,            (PostgreSQL + Auth)
        │    schedule, community, notifications)
        │
        └──────────────────────────────────────▶ Express server (Railway)
                                                  └─▶ Google Gemini API
                                                      (AI-тьютор, проверка ДЗ)
```

## Слои приложения

| Слой | Папка | Что делает |
|------|-------|-----------|
| Точка входа | `App.tsx` | Провайдеры + роутинг |
| Страницы | `views/` | Полные экраны по роутам |
| Компоненты | `components/` | Переиспользуемые блоки |
| Состояние | `contexts/` | Глобальные данные |
| Хуки | `hooks/` | Логика, повторно используемая в компонентах |
| Сервисы | `services/` | Запросы к Supabase и API |
| Типы | `types.ts` | Все TypeScript интерфейсы |
| Данные | `constants.ts` | Мок-данные для разработки |
| Сервер | `server.js` | Express: AI-прокси + раздача статики |

## Порядок провайдеров (App.tsx)

```
BrandingProvider        ← брендинг школы (самый внешний)
  └── AuthProvider      ← пользователь и роль
        └── ContentProvider  ← курсы и прогресс
              └── ToastProvider  ← уведомления
                    └── AppContent (BrowserRouter + Routes)
```

## Роутинг и защита

```
/              → Dashboard           (открыт для всех, включая Guest)
/courses       → CourseDetail        [ProtectedRoute]
/profile       → Profile             [ProtectedRoute]
/schedule      → Schedule            [ProtectedRoute]
/community     → Community           [ProtectedRoute]
/settings      → Settings            [ProtectedRoute]
/notifications → Notifications       [ProtectedRoute]
/users/:userId → UserPublicProfile   [ProtectedRoute]
/admin         → AdminPanel          [StaffRoute — только ADMIN]
```

`ProtectedRoute` → редирект на `/` если isGuest
`StaffRoute` → редирект на `/` если роль не ADMIN

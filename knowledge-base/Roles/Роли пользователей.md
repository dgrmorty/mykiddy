# Роли пользователей
#roles #auth

| Роль | Enum | Доступ |
|------|------|--------|
| Student | `Role.STUDENT` | Все защищённые роуты, курсы, профиль |
| Parent | `Role.PARENT` | Просмотр прогресса ребёнка |
| Teacher | `Role.TEACHER` | Работа с контентом |
| Admin | `Role.ADMIN` | Всё + AdminPanel |
| Guest | `Role.GUEST` | Только Dashboard (readonly) |

## Определение роли Admin
Через `VITE_ADMIN_EMAILS` в env (список через запятую).
Если переменная не задана — есть fallback email в коде (убрать в проде!).

## Guards в App.tsx
```
ProtectedRoute → isGuest? → redirect /
StaffRoute     → role !== ADMIN? → redirect /
```

# Система прогрессии
#gamification #xp

## Формула
```
XP_PER_LEVEL = 500
level = floor(xp / 500) + 1
progressInLevel% = (xp % 500) / 500 * 100
```

## Файл
`progression.ts` — две функции: `levelFromXp`, `xpLevelProgressPercent`

## Начисление XP
- Завершение урока → +XP
- Стрик (ежедневный вход) → бонус
- Достижения → бонус

## Хранение
В `profiles`: колонки `xp`, `level`, `streak_current`, `streak_longest`

# Контексты
#contexts #react

## AuthContext
**Файл:** `contexts/AuthContext.tsx`
```typescript
useAuth() → { user, isGuest, isLoading, signIn, signOut }
```
- Загружает профиль из Supabase при старте
- Определяет роль по email (admin) или из profiles
- `GUEST_USER` из constants.ts — фоллбэк до загрузки

## ContentContext
**Файл:** `contexts/ContentContext.tsx`
```typescript
useContent() → { courses, progress, markLessonComplete, ... }
```
- Загружает курсы, модули, уроки из Supabase
- Кэширует прогресс пользователя
- В dev-режиме может использовать MOCK_COURSES

## BrandingContext
**Файл:** `contexts/BrandingContext.tsx`
```typescript
useBranding() → { branding }
```
- Загружает настройки из таблицы `settings`
- Цвета, логотип, название школы

## ToastContext
**Файл:** `contexts/ToastContext.tsx`
```typescript
useToast() → { showToast }
```
- Глобальные уведомления (успех, ошибка, инфо)

## Правило
Не делай прямые запросы к Supabase в компонентах — используй контексты!

# Антипаттерны
#rules #antipatterns

> Это список ошибок которые нельзя допускать в коде проекта.

1. **❌ Gemini напрямую с фронта** → только через `/api/*` сервера
2. **❌ Новый Supabase client** → используй `services/supabase.ts`
3. **❌ Бизнес-логика в компонентах** → выноси в хуки/контексты
4. **❌ Проверка роли только на фронте** → RLS в Supabase обязателен
5. **❌ Убирать rate limiting** с AI-эндпоинтов
6. **❌ Новые типы вне types.ts** → все интерфейсы только там
7. **❌ Прямой fetch к Supabase в компоненте** → используй контексты
8. **❌ Трогать android/ без задачи** по мобайлу
9. **❌ Новые UI-библиотеки** (MUI, Chakra и т.д.) без запроса
10. **❌ Inline-стили** там где можно Tailwind

# Слабые места
#security #issues

Из файла `WEAK_SPOTS.md` в репо:

## Безопасность
- **Admin только на фронте** — RLS в Supabase для admin-таблиц критичен
- **Supabase URL/key в коде** — лучше через `VITE_SUPABASE_*` env
- **Fallback admin email** в коде — убрать в проде, использовать `VITE_ADMIN_EMAILS`

## Надёжность
- **Нет общего rate limit** — только AI-эндпоинты защищены
- **Много console.log** в коде — в проде логировать в внешний сервис

## Архитектура
- **Один конфиг для всех окружений** — использовать `VITE_API_BASE_URL`
- **Миграции SQL удалены** — схема только в БД, нужны бэкапы

## Что уже хорошо ✅
- Санитизация от prompt injection в AI-чатах
- CORS ограничен доменами
- Лимиты AI по JWT и по дням
- Gemini API ключ только на сервере

# Окружение и деплой
#devops #env

## Переменные окружения

| Переменная | Где задаётся | Назначение |
|-----------|-------------|-----------|
| `VITE_SUPABASE_URL` | Vercel | URL проекта Supabase |
| `VITE_SUPABASE_ANON_KEY` | Vercel | Публичный anon ключ |
| `VITE_ADMIN_EMAILS` | Vercel | Emails админов через запятую |
| `VITE_API_URL` | Vercel | URL Express-сервера на Railway |
| `API_KEY` | Railway | Gemini API Key (только сервер!) |
| `PORT` | Railway | Порт Express-сервера |

## Деплой фронта — Vercel
- Автодеплой из `main` ветки
- Билд команда: `npm run build` (Vite)
- Output: `dist/`

## Деплой сервера — Railway
- `server.js` — Express
- `Procfile`: `web: node server.js`
- `nixpacks.toml` — конфиг сборки
- `railway.json` — конфиг Railway

## Локальная разработка
```bash
cp .env.example .env   # заполни переменные
npm run dev            # Vite dev server
npm run server         # Express сервер отдельно
```

## Android APK
```bash
npm run apk
```

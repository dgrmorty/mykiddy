# Антипаттерны и осторожности

#code #quality

---

## Безопасность

- Не вставлять **ключ Gemini** или **service role** Supabase во фронт или в git.
- Не отключать **rate limit** на `/api/*` без явной задачи и ревью.
- Не ослаблять **RLS** «на всю таблицу» ради одного экрана — предпочтительно **узкий RPC** SECURITY DEFINER.

---

## Архитектура

- Не создавать второй экземпляр Supabase-клиента.
- Не звать Gemini с браузера минуя `server.js`.
- Избегать **пустых `catch`** без лога (особенно стрики и витрина).

---

## UX / данные

- Не ломать **`mergePreserveAvatar`** и серийность **`profileFetchSerialRef`** в Auth без понимания гонок.
- Курсы — через **`contentService`**, не дублировать гигантские select в каждом view.

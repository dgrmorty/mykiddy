## Безопасность

- **Админ по email:** зависимость от `VITE_ADMIN_EMAILS`; при пустой переменной возможны fallback в коде — проверять перед продом.
- **RLS:** фронт не заменяет политики; любой эндпоинт с anon/authenticated ключом должен быть покрыт политиками (см. [[Architecture/RLS-RPC-и-безопасность]]).
- **CORS / ключи AI** — только на сервере.

## Документация

- Файл **`cursor-rules/PROJECT.md`** мог содержать устаревшие имена API (**`/api/chat`**); актуально: **`/api/ai-tutor`**, **`/api/check-homework`**. Дублируется в [[Architecture/Сервер-Express-и-AI]].

# Как дать себе вкладку «Управление» (админ)

Роль админа берётся из двух мест (достаточно одного):

1. **Переменная при сборке** `VITE_ADMIN_EMAILS=knazar002@gmail.com`  
   Работает только если она задана **в момент** `npm run build` (например в Railway в Variables).

2. **Поле в базе Supabase** — столбец `role` в таблице `profiles`.  
   Если там стоит `Admin` или `admin`, вкладка «Управление» показывается независимо от переменной.

---

## Быстрый способ (через Supabase)

1. Зайди на **https://supabase.com** → свой проект.
2. Слева: **Table Editor** → таблица **profiles**.
3. Найди строку с твоим email **knazar002@gmail.com** (или по своему `id` из Auth).
4. В колонке **role** поставь значение **Admin** (с большой буквы можно).
5. Сохрани (галочка / Save).
6. В приложении: выйди из аккаунта и зайди снова **или** обнови страницу (F5).

После этого у аккаунта knazar002@gmail.com должна появиться вкладка «Управление».

---

## Если хочешь задавать админов через переменную (при деплое)

- В **Railway**: в настройках сервиса → **Variables** добавь  
  `VITE_ADMIN_EMAILS` = `knazar002@gmail.com`  
  (несколько адресов через запятую без пробелов).
- Важно: переменные с префиксом **VITE_** подставляются **при сборке**. После добавления переменной сделай **Redeploy**, чтобы сборка прошла заново с этой переменной.

Если и переменная задана, и в `profiles.role` стоит Admin — оба варианта дают доступ; можно использовать любой.

# Что нужно сделать после внедрения исправлений аудита

Все изменения по файлу **AUDIT_REPORT.md** внесены в код. Ниже — что **обязательно нужно сделать тебе** (настроить окружение, Supabase, деплой).

---

## 1. Переменные окружения

Скопируй `.env.example` в `.env` и заполни значениями.

### На локальной разработке

- **VITE_API_URL** — для `npm run dev` можно не ставить (будет использоваться proxy на localhost:3000 из vite.config) или поставить `http://localhost:3000`.
- **VITE_SUPABASE_URL**, **VITE_SUPABASE_ANON_KEY** — из дашборда Supabase: Project Settings → API.
- **VITE_ADMIN_EMAILS** — твой email (или несколько через запятую), чтобы заходить в админку.
- **API_KEY** — ключ Google AI (Gemini). Получить: https://aistudio.google.com/apikey
- **SUPABASE_URL**, **SUPABASE_ANON_KEY** — те же, что и для клиента (сервер ходит в Supabase с JWT пользователя).

### На продакшене (Railway / другой хостинг)

- Задай **все** переменные из `.env.example` в настройках сервиса.
- **VITE_*** задаются на этапе **сборки** (build), не в runtime. В Railway: добавь их в Variables и пересобери проект, чтобы они попали в сборку.
- **Обязательно** задай **VITE_ADMIN_EMAILS** в продакшене — иначе админ-панель будет недоступна (в коде убран fallback).

---

## 2. Supabase: RLS и RPC

Миграция **`20260418120000_security_hardening`** уже применена к связанному проекту: админ в БД = **`profiles.role = 'Admin'`** (функция `is_admin_user()`), `increment_xp` только **`x_val = 50`**, отозваны `EXECUTE` у `anon`/`PUBLIC` для RPC, убрано чтение чужих **ДЗ/прогресса** всеми учениками, сжаты политики **profiles** и **лайков**.

### Админы в БД

- Для каждого администратора в таблице **`profiles`** должно быть **`role = 'Admin'`** (регистр как в приложении: `Admin`). Иначе RLS не даст править курсы/расписание, даже если в UI админка открыта по email.
- В Dashboard: **Table Editor → profiles** или SQL: `update profiles set role = 'Admin' where id = '<uuid>';`

### RPC `increment_xp`

- Принимает только **`50`**; иначе ошибка. Начисление из кода уже с `x_val: 50`.

### Индексы (рекомендуется)

Создай индексы по аудиту:

- **user_progress**: `(user_id)`, `(lesson_id)`, уникальный `(user_id, lesson_id)`.
- **homework_submissions**: `(user_id, lesson_id)`.
- **ai_usage**: уникальный `(user_id, date)`.
- **profiles**: по `xp DESC` для лидерборда.

### Подтверждение email (по желанию)

В Supabase: Authentication → Providers → Email включи **Confirm email**. Тогда после регистрации пользователь получит письмо; в приложении уже показывается сообщение «Проверьте почту».

## 3. Восстановление пароля

В Supabase: Authentication → URL Configuration задай **Site URL** (например `https://mykiddy-production.up.railway.app`) и при необходимости **Redirect URLs** для сброса пароля. После этого ссылка «Забыли пароль?» в AuthModal будет вести на письмо с восстановлением.

---

## 4. Что скинуть или проверить

- **Файл с переменными** — не скидывай `.env` с реальными ключами. Скинь только список **имен** переменных, которые ты реально используешь (или убедись, что они совпадают с `.env.example`).
- **Supabase**: скрин или экспорт политик RLS и определение RPC `increment_xp` — если хочешь, чтобы я проверил логику доступа и начисления XP.
- **Проверка после деплоя**:
  - Вход / регистрация.
  - Восстановление пароля (письмо приходит, ссылка открывается).
  - Дашборд: курсы и прогресс подгружаются, события из `schedule_events` отображаются.
  - Курсы: открытие урока, видео, ДЗ, проверка ДЗ через ИИ, кнопка «Повторить» при ошибке загрузки.
  - Админка: доступ только под VITE_ADMIN_EMAILS; загрузка каталога одним запросом (без N+1).
  - Офлайн: при отключении сети появляется жёлтый баннер «Нет соединения».

- [ ] Создан `.env` из `.env.example`, все значения заполнены.
- [ ] В продакшене заданы переменные сборки **VITE_*** и переменные сервера (API_KEY, SUPABASE_*, PORT, при необходимости CORS_ORIGIN).
- [ ] В продакшене задан **VITE_ADMIN_EMAILS**.
- [ ] В Supabase включён и настроен RLS для всех таблиц.
- [ ] RPC **increment_xp** использует **auth.uid()**.
- [ ] При необходимости созданы индексы (user_progress, homework_submissions, ai_usage, profiles).
- [ ] В Supabase настроены Site URL и при необходимости Redirect URLs для сброса пароля.
- [ ] После деплоя проверены: вход, восстановление пароля, курсы, ДЗ, админка, офлайн-баннер.

После этого приложение будет соответствовать внедрённым по аудиту исправлениям и готово к использованию с учётом твоей инфраструктуры.
